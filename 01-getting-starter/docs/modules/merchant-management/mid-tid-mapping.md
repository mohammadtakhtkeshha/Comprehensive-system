# نگاشت پذیرنده به MID/TID

## Merchant ID (MID)

هر پذیرنده یک شناسه یکتا به نام Merchant ID دریافت می‌کند که در تمام تراکنش‌ها استفاده می‌شود.

### ویژگی‌های MID
- شناسه یکتا و غیرقابل تکرار
- قابل استفاده در تمام تراکنش‌ها
- مرتبط با اطلاعات پذیرنده

### تخصیص MID
- به صورت خودکار در زمان فعال‌سازی پذیرنده
- قابل تغییر در شرایط خاص
- ثبت در سامانه مرکزی

## Terminal ID (TID)

هر ترمینال یک شناسه یکتا به نام Terminal ID دارد که در تراکنش‌ها استفاده می‌شود.

### ویژگی‌های TID
- شناسه یکتا برای هر ترمینال
- مرتبط با MID پذیرنده
- مرتبط با اطلاعات ترمینال

### تخصیص TID
- در زمان تحویل ترمینال
- مرتبط با MID پذیرنده
- ثبت در سامانه

## نگاشت MID-TID

### روابط
- یک MID می‌تواند چندین TID داشته باشد
- هر TID فقط به یک MID تعلق دارد
- TID می‌تواند بین MIDها منتقل شود

### مدیریت نگاشت
- افزودن TID جدید به MID
- انتقال TID بین MIDها
- حذف TID از MID
- مشاهده لیست TIDهای یک MID

## عملیات‌ها

### افزودن TID جدید
1. انتخاب پذیرنده (MID)
2. انتخاب ترمینال
3. تخصیص TID
4. ثبت و فعال‌سازی

### انتقال TID
1. انتخاب TID
2. انتخاب MID مقصد
3. تایید انتقال
4. ثبت تغییرات

### حذف TID
1. انتخاب TID
2. بررسی وابستگی‌ها
3. تایید حذف
4. ثبت تغییرات

## مدیریت همزمانی

در هنگام نگاشت MID-TID، سیستم اطمینان حاصل می‌کند که:
- هر TID فقط به یک MID تعلق دارد
- اطلاعات همزمان به‌روز می‌شود
- تغییرات در سامانه مرکزی ثبت می‌شود

## گزارش‌گیری

از این بخش می‌توانید گزارش‌های زیر را دریافت کنید:
- فهرست MIDها و TIDهای مرتبط
- تاریخچه تغییرات نگاشت
- TIDهای بدون MID
- MIDهای بدون TID
